# Earthquake Notification System (ENS)

## 1. პროექტის მიმოხილვა

Earthquake Notification System (ENS) წარმოადგენს რეალურ დროში მოქმედ მიწისძვრების შეტყობინებების პლატფორმას, რომლის მიზანია სეისმური მოვლენების ავტომატური დამუშავება და Android და iOS მომხმარებლებისთვის მყისიერი შეტყობინებების მიწოდება.

სისტემა ინტეგრირებულია SeisComP-ის სეისმური დამუშავების ინფრასტრუქტურასთან და უზრუნველყოფს მიწისძვრის მოვლენების მიღებას, დამუშავებას, შენახვასა და გავრცელებას.

---

# 2. პროექტის მიზანი

პროექტის ძირითადი მიზანია:

- მოსახლეობის დროული ინფორმირება მიწისძვრის შესახებ;
- მიწისძვრების მონაცემების თითქმის რეალურ დროში გავრცელება;
- სეისმური მონაცემების ცენტრალიზებული მართვა;
- შეტყობინებების პერსონალიზაცია მომხმარებლის პარამეტრების მიხედვით;
- საიმედო, უსაფრთხო და მასშტაბირებადი ინფრასტრუქტურის შექმნა.

---

# 3. ძირითადი ფუნქციონალი

## მომხმარებლებისთვის

- რეგისტრაცია და ავტორიზაცია;
- მიწისძვრების სიის ნახვა;
- მიწისძვრის დეტალური ინფორმაციის ნახვა;
- Push შეტყობინებების მიღება;
- შეტყობინებების პარამეტრების მართვა;
- რამდენიმე მოწყობილობის მხარდაჭერა.

---

## ადმინისტრატორებისთვის

- მომხმარებლების მართვა;
- მომხმარებლის უფლებების მართვა;
- სისტემური შეტყობინებების გაგზავნა;
- სტატისტიკისა და მონიტორინგის მონაცემების ნახვა;
- სისტემის ადმინისტრირება.

# 5. სისტემის ძირითადი მოთხოვნები

## ფუნქციური მოთხოვნები

- მომხმარებლის რეგისტრაცია და ავტორიზაცია;
- მიწისძვრის მოვლენების დამუშავება;
- Push შეტყობინებების გაგზავნა;
- მომხმარებლის პარამეტრების მართვა;
- ადმინისტრაციული ფუნქციები;
- მონიტორინგი და ლოგირება.

---

## არაფუნქციური მოთხოვნები

- მაღალი ხელმისაწვდომობა (High Availability);
- მასშტაბირებადობა (Scalability);
- უსაფრთხოება (Security);
- მდგრადობა (Fault Tolerance);
- მონაცემთა სარეზერვო ასლები;
- მაღალი წარმადობა (Performance).

# 10. პროექტის მიზნობრივი შედეგი

პროექტის საბოლოო მიზანია საქართველოში თანამედროვე, საიმედო და მასშტაბირებადი მიწისძვრების შეტყობინებების პლატფორმის შექმნა, რომელიც უზრუნველყოფს სეისმური მოვლენების სწრაფ გავრცელებას და მომხმარებლების დროულ ინფორმირებას.
